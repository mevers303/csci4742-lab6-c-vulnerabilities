Introduction

Memory corruption vulnerabilities pose significant security risks, often leading to unauthorized memory access, crashes, or remote code execution. This section explores common memory corruption issues, including buffer overflows, use-after-free, uninitialized memory usage, and integer overflow.

By implementing these vulnerabilities in a controlled environment, we will analyze their impact using dynamic analysis tools such as Valgrind and AddressSanitizer, and perform static analysis with FlawFinder to detect potential security flaws.

📌 Task 1: Implement and Test a Vulnerable C Code (10 pts)

🔹 Step 1: Develop a Vulnerable C Program

Develop the following C program on your Kali Linux attack machine. This program contains multiple security vulnerabilities, including buffer overflows, dangling pointers, uninitialized memory usage, and integer overflows. You will use this code to analyze and understand common memory-related exploits.

📌 vulnerable_program.c:

#include <stdlib.h>
#include <stdio.h>
#include <string.h>
#include <time.h>
#include <limits.h>  // Required for integer limits

void overRun(void) {
    int *x = malloc(10 * sizeof(int));
    x[10] = 0;  // Buffer overrun
}

void randStringGen(int x, char* c) {
    srand(time(NULL));
    for (int i = 0; i < x - 1; ++i) {
        *c = 'A' + (rand() % 26);
        c++;
    }
    *c = '\0';
}

void bufferUnder(void) {
    char buffer[256];
    char *c = malloc(255 * sizeof(char));
    randStringGen(255, c);
    strcpy(buffer, c);  // Possible buffer overflow
    printf("%s\n", buffer);
    free(c);
}

void danglingPtr(void) {
    int *x;
    int *y = malloc(10 * sizeof(int));
    x = y;
    free(y);  // x is now a dangling pointer
    int t = x[2];  // Accessing freed memory
    printf("Dangling pointer value: %d\n", t);
}

void unInitializedPtr(void) {
    char *buffer;
    char *c = malloc(10 * sizeof(char));
    randStringGen(10, c);
    strcpy(buffer, c);  // Using an uninitialized pointer
    printf("%s\n", buffer);
    free(c);
    free(buffer);
}

void bufferOver(void) {
    char buffer[256];
    char *c = malloc(260 * sizeof(char));
    randStringGen(260, c);
    strcpy(buffer, c);  // Buffer overflow
    printf("%s\n", buffer);
    free(c);
}

// New: Integer Overflow Vulnerability
void integerOverflow(void) {
    int a = INT_MAX;  // Max signed int value
    int b = 1;
    int result = a + b;  // Causes overflow
    printf("Integer Overflow: %d + %d = %d\n", a, b, result);
}

int main(int argc, char**argv) {
    if (argc != 2) {
        return 0;
    }
    int x = atoi(argv[1]);  // Convert input to integer

    if (x == 1) {
        overRun();
    } else if (x == 2) {
        unInitializedPtr();
    } else if (x == 3) {
        danglingPtr();
    } else if (x == 4) {
        bufferUnder();
    } else if (x == 5) {
        bufferOver();
    } else if (x == 6) {
        integerOverflow();
    }

    return 0;
}

By default, Valgrind is packaged in Kali’s main repository, so an apt-get install valgrind should typically work. If you are getting a “not in repository” error, it usually means either your repositories are misconfigured or out-of-date. Below are the main steps to fix it:

2. Install Valgrind

You should see a version number (e.g., valgrind-3.20.0).

🔹 Step 1: Investigate the overRun Function

The overRun function creates an array of 10 integers using malloc, but then attempts to write to index 10, which is out of bounds (since valid indices range from 0 to 9).
